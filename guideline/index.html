コーディングガイドライン

本ガイドラインは、WEBサイト制作を行う際の記述ルールなどを定めたものです。

ガイドラインについて

目的

本ガイドラインは、新規ページの作成や既存ページの修正・更新作業の際に、クオリティーの統一とメンテナンス性の向上を目的に作成しています。
コーディングを行うスタッフは、ガイドラインがある意味と内容をよく理解し、ガイドラインに則ったコーディングを行ってください。

命名規則

HTMLファイル、画像ファイル及びCSS（ID及びCLASSセレクタ）の名前の付け方は以下を基本ルールとします。

  1. 半角英数字のみを使用。
  2. 記号は「-」（ハイフン）、「_」（アンダースコア）のみ使用。
  3. 全角スペース、半角スペースは使用しないでください。
  4. 機種依存文字の使用を禁止します。
  5. 数字や記号から開始しないでください。
  6. 単語の文字数の省略は、基本的に行わないのが原則です。

NG例）

テキストファイル.gif
\text.gif
img　text
500px

HTMLファイル

  • 何のページか推測出来る英単語や慣例的に使われている名前を付けてください。
  • ファイル名が長くなりすぎてしまう場合は、単語を1～2個に絞ること。
  • その場合「_」（アンダーバー）で単語をつなげてください。

id・class名の付け方

id・class名はその名前からページ上のどこに使われているのか推測出来るものにしてください。
例えば#headerというid名の場合、ページ上のヘッダー部分に使われているだろうということが推測できます。
なお、コンテンツ特有のid・class名が必要な場合は、コンテンツ名を英語に翻訳したid・class名になります。

このとき、文字数が多すぎるとコードが読みにくくなるため、特徴的な単語を抜き出しても構いません。 英単語が複数にわたる場合は、「_」（アンダーバー）で単語をつなげてください。

例）

service point information →　service_point_info

名前の省略（例外処置）

命名規則の基本ルールに「 単語の文字数の省略は、基本的に行わないのが原則です。」と記述しています。 しかし省略形が広く一般的に認知されており、使用しても不自然ではない場合は積極的に利用します。 また、一般的に省略されることはあまりないが、冗長すぎてファイル管理が複雑になってしまう場合は以下の方法で省略してください。

省略形の例）

  • ボタン button → btn（ボタンに関しては常にbtnで記述）
  • テキスト text → txt（テキストに関しては常にtxtで記述）
  • 背景 background → bg （背景に関しては常にbgで記述）
  • ナビゲーション　navigation → nav
  • 紹介　introduction → intro
  • 確認 confirmation → confirm
  • お知らせ　information → info
  • contents → conts

id・class名の使い分け

id

  1. ヘッダー・フッター・メインコンテンツ・サイドバーなど大枠のレイアウトワイヤースタイル
  2. 恒久的にページで唯一となるコンテンツのスタイル
  3. ページ内で「章」にあたるコンテンツの最上位階層の要素
  4. ページ内アンカーの対象となる要素

class

  1. サイト共通のスタイル
  2. IDに当てはまらない全てのスタイル

HTML

HTMLの記述は以下を基本ルールとします。

  1. 通常「”」（ダブルクォート）を使います。属性値に対して「’」（シングルクォート）を使用しないでください。
  2. 省略できる閉じタグを省略しないでください。(例えば</li>や</body>)。
  3. エンプティ要素（img ・ br ・ meta ・ linkなど）は最後を閉じないでください。
  4. Alt属性は必ず記述してください。適切な文言がない場合はNull値とします。
  5. target属性は_blank指定とします。

マークアップ

  • 文書構造的に正しいマークアップを行なってください。
  • タグ（divやspanなど）を過剰に使用したコードを書かないでください。
  • コンテンツやテキストの増減で崩れない構造にしてください。

マークアップのコツ

マークアップをするうえで最大の問題となるものがデザインです。
デザインの表現は、ときに文書構造を無視し見た目を優先します。
そういったデザインを見たままマークアップしてしまうと誤った文書構造になっていまいます。
そうならないために、マークアップの際にはデザインを無視し、目次は何か、見出しとそれに対応する文章は何か、文章は段落なのか箇条書きなのかを考えながら1サイトを1冊の本にするイメージで行なうことで正しい文書構造を考えながらマークアップすることができます。

文字コード

素早く、そして簡単に適切なレンダリングがされるよう文字エンコードを宣言しましょう。こうすることで、特殊文字を使用しなくても良くなります。

ユーザビリティ、アクセシビリティ、SEOの観点から適切なものを設定します。

Internet Explorerのレンダリングモード

headタグに以下のコードを記述し、互換表示なしの最新モードでレンダリングされるようにしてください。

<meta http-equiv="X-UA-Compatible" content="IE=edge">

最近のInternet Explorerでは表示を古いバージョンのものにする互換表示機能があります。 この機能をユーザー側で意図せず有効にしてしまい、表示が崩れることがあるため、互換表示機能をOFFにするために上記のコードを使用します。

  • HTML ファイル内は基本 HTML コードのみの記述とし、CSS や JavaScript などについては外部ファイルにします。

  • jsの読み込みには、src,type,charsetの3属性を指定します。

CSS

CSSの記述は以下を基本ルールとします。 スタイルシートは複雑化しやすいため、統一されたルールに則った構造設計や記述を行ってください。

  1. CSSの定義は原則として外部CSSに記述してclass及びidで呼び出します。
  2. HTMLに直接記述する事は原則として禁止します。
  3. Charsetの記述はHTMLと同じ「UTF-8」を設定。
  4. 全ての宣言をセミコロンで終わらせてください。
  5. 値が「0」なら単位を省略します。
  6. 小数点以下の値を指定する時、1の位が0なら省略します。（-0.5pxではなく-.5px）
  7. line-height に単位はつけないでください。
  8. 画面の装飾は基本的にCSSでのみ行い、HTMLでは行いません。

セレクタの書き方

  • 共通スタイルはどこでも使えるように、IDを含めないようにしてください。
  • 共通スタイル以外は必ずIDから始めてください。
  • 全称セレクタは使用禁止とします。
  • HTMLに出てくるコンテンツの順番とセレクタの順番を同じにします。
  • セレクタはできるだけ短くするように努めてください。

宣言の順序

CSSのプロパティの記述順を以下のように定めます。

  1. 位置関係を定義するプロパティ
    display position float など
  2. ボックスモデルを定義するプロパティ
    width height border など
  3. 文字の体裁やフォントを定義するプロパティ
    font color text-align など
  4. 見た目に関するプロパティ

※上記4つのブロックが確立されていれば、その中でのプロパティの順番は問いません。

例）

.declaration-order {
/* Positioning */
	position: absolute;
	top: 0;
	right: 0;
	bottom: 0;
	left: 0;
	z-index: 100;

/* Box-model */
	display: block;
	float: right;
	width: 100px;
	height: 100px;

/* Typography */
	font: normal 13px "Helvetica Neue", sans-serif;
	line-height: 1.5;
	color: #333;
	text-align: center;

/* Visual */
	background-color: #f5f5f5;
	border: 1px solid #e5e5e5;
	border-radius: 3px;

/* Misc */
	opacity: 1;
}

フォント周りの単位指定について

font-size

上からpx,remの順に指定してください

例）

html { font-size: 62.5%;} /* font-size 62.5% = 10px = 1rem */
body {
	font-size: 10px;
	font-size: 1.0rem;
}
remはrootであるHTMLのフォントサイズを基準とした相対値指定となるため、HTMLを10pxとしておけば14pxのときは1.4rem、26pxのときは2.6remといった形で計算せずに指定したいpxを相対値で指定できるため簡単に扱うことが出来ます。

line-height

単位なしで指定してください。

なぜ単位をつけないのか

  • line-heightでは、font-sizeとline-heightが掛け合わされた値が一行の高さとして指定されます。
    例えば、p { font-size: 14px; line-height: 1em; }といったスタイルの場合には一行の高さが14pxとなります。 （14*1=14）
  • 単位を指定した場合には親要素の一行の高さを子孫要素に継承し、
    指定しない場合にはline-heightの値を子孫要素に継承します。
以上を踏まえて、単位を指定した場合を考えてみると、 例えば単位をつけて一行の高さが14pxで継承されてしまうと、子孫要素のfont-sizeが26pxだった場合に行間が14pxとなり、テキストの方が大きな状態となります。こうなると、一行目と二行目のテキストが重なってしまいます。単位を指定しない場合は、変化するフォントサイズに適切な行間を計算で出すことができます。

リセットについて

リセットcssを外部リンクとして読み込みます。 <link>の中でも、最も上の位置（最初）に記述してください。

ベンダープリフィックスについて

ベンダープリフィックスを付与する時は、値が揃う位置までインデントして揃えてください。こうする事で、複数行の編集が容易になります。

例）

/* Prefixed properties */
.selector {
	-webkit-box-shadow: 0 1px 2px rgba(0,0,0,.15);
          box-shadow: 0 1px 2px rgba(0,0,0,.15);
}

また、順番は以下のようになるよう努めてください。

  1. -webkit-
  2. -moz-
  3. -ms-
  4. -o-
  5. ベンダープレフィックスなしのプロパティ

※そのサイトでサポートするブラウザに合わせて１～４は削除してください。

CSSハックについて

CSSハックとは、Webブラウザ間のCSSの解釈の違いやバグを振り分けの条件とし、特定のブラウザに対しスタイルを適用、または非適用とする手法の事をいいます。 基本的に利用しないのを原則とします。

clearfixの書き方

以下のコードを使用してください。

.clearfix:after {
	display: block;
	clear: both;
	content: "";
}
　
　

JavaScript

JavaScriptの記述は以下を基本ルールとします。

  1. HTMLやCSSで出来ないかを考えてから使用してください。
  2. 何をしているものかコメントを付けてください。
  3. 動作チェックの他に、見ただけでは分からないようなエラーがでていないか各ブラウザのデベロッパーツールのコンソールで確認してください。
  4. 外部JavaScriptファイルの読み込みは、動作に問題がない限りは </body> の直前で行い、問題がある場合には <head> … </head> 内で読み込んでください。
  5. 複数のページで使うことが想定されるものはcommon.jsにまとめてください。
  6. jQueryプラグインやその他ライブラリは軽量化（ミニファイ）されたものを使用してください。
	<script type="text/javascript" src="/common/js/jquery.js"></script>
	<script type="text/javascript" src="/common/js/rollover.js"></script>
</body>

ライブラリについて

対応ブラウザがIE9以上であればjQuery 3.X系を使用してください。 対応ブラウザにIE8以下が含まれる場合はjQuery 1.12 の最新版を使用してください。

スマホコーディング

Viewportの設定

リキッドデザイン（ユーザーによる表示倍率の操作不可）のときは回転しても表示倍率や文字サイズが変更されないよう以下の設定をしてください。

<meta name="viewport" content="width=device-width, initial-scale=1.0, minimum-scale=1.0, maximum-scale=1.0, user-scalable=no">
  • content="width=device-width /*端末画面の幅に合わせる*/
  • initial-scale /*初期のズーム倍率*/
  • minimum-scale /*最小倍率*/
  • maximum-scale/*最大倍率*/
  • user-scalable /*ズームの操作 yes 許可する （初期値）/ no 許可しない*/

User-scalableをnoにしないと回転したときに表示倍率が変わってしまいますが、ユーザーが拡大・縮小操作をできないといったデメリットもあるため仕様を確認して下さい。

文字サイズ自動調節機能

AndroidのChromeブラウザで閲覧した際に部分的にフォントサイズが意図せず大きくなってしまうことがあります。以下の解決方法を記述してください。

body {
	-webkit-text-size-adjust: 100%;
}
/*pのテキストが大きい場合*/
p {
	max-height: 100%;
}

画像について

スマホコーディング/レスポンシブウェブデザインを行うにあたり、 PC用の画像とSP（スマホ）用の画像が必要になります。 ファイル名は以下のように区別してください。 また、PCサイズ＝SPの二倍サイズ…のように、一枚のサイズを切り替えて使用する場合は 通常の命名で構いません。

例）

  • image_h1_pc.jpg/* PC */
  • image_h1_sp.jpg/* SP */
  • image_h1.jpg/* PC、SP共通して使用する場合 */

Ratina（高解像度）ディスプレイへの対応方法

デザインデータが640pxのサイズで作られている場合は、高解像度ディスプレイ向けのデザインになっていますので以下のルールでスライスと配置をしてください。

  • 画像サイズは余白を含めてよいので、必ず偶数にしてください。
  • imgタグで配置するときはwidthを画像サイズの2分の1の値で指定すること。
    その際heightは指定しないでください。
  • 背景に指定するときはbackgrond-sizeで2分の1の値を指定すること。
    Retina用にスライスした画像のファイル名は以下のようにしてください。

例）

  • image_h1_sp.jpg/* 通常の画像用指定　*/
  • image_h1_sp@2x.jpg/* Retinaディスプレイ用高解像度画像指定　*/

レスポンシブウェブデザイン

コーディングする順番

レスポンシブウェブデザインでは、 モバイルファーストでコーディングを行ってください。

例）

例えば、PCのデザインで左右に写真があるデザインがあった際、PCはどちらを先にマークアップしたとしても、CSSで調整が可能です。（基本的には左が先ですが）
しかしスマートフォンでは、その写真が上下に並ぶデザインになる可能性があります。その場合、マークアップは必ず上から順に並べてマークアップする必要があります。PCを基準にしたがゆえの問題を防ぐために、スマートフォンを基準に考えます。

ブレイクポイントの基準・書き方

ブレイクポイントは下記を基準に設定します。 ただし、あくまで基準のためデザインや対応デバイスによっては変更します。

  • スマートフォン
    0～599px
  • タブレット
    600px～959px
  • PC
    960px以上

メディアクエリを記述する箇所

スマートフォン用のCSSを記述し終えた下に、タブレット用のCSS、PC用のCSSを記述してください。
デフォルトでスマートフォン用のスタイルを定義し、タブレット／PC用のスタイルをMedia Queriesを使って上書きしていくモバイルファーストな方法です。

要素ごとに調整を行うよりも、スマートフォンの調整やタブレットの調整といったデバイスごとの調整を行うことの方が多いため、デバイスごとにまとめて記述するようにします。
/* ここにスマートフォン用のCSS *

@media only screen and (min-width: 600px and max-width: 959px) {
/* ここにタブレット用のCSS */
}
				
@media only screen and (min-width: 960px) {
/* ここにPC用のCSS */
}

フォントサイズ切り替え

フォントサイズ切り替えは以下2パターンとします。
LPなど小～中規模サイトはパターン1、
複数階層にわたる大規模サイトはパターン2を使用してください。

パターン1

htmlのフォントサイズのみ、メディアクエリーで調節します。

remの値は、html要素に対して相対的です。
つまり、htmlが基準となっているため、htmlのフォントサイズを大きくすれば、 他要素のサイズも大きくなります。（比率は変わりません）
html{
	font-size:10px; /* font-size 1.0rem = 10px */
}
body {
	font-size: 16px; 
	font-size: 1.6rem;  /* font-size 1.6rem = 16px */
}
/* Typography
–––––––––––––––––––––––––––––––––––––––––––––––––– */
h1 {
	font-size: 4.0rem;
}
h2 {
	font-size: 3.6rem;
}
h3 {
	font-size: 3.0rem;
}
h4 {
	font-size: 2.4rem;
}
h5 {
	font-size: 1.8rem;
}
h6 {
	font-size: 1.5rem;
}
/* Larger than phablet */
@media (min-width: 600px) {
html{
	font-size:20px; /* タブレットではSP版の二倍の大きさで表示される */
}
}

パターン2

メディアクエリーの画面幅ごとに、要素のフォントサイズを設定していきます。

html{
	font-size: 62.5%; /* font-size 62.5% = 10px = 1rem */
}
body {
	font-size: 1.6px; 
	font-size: 1.6rem;  /* font-size 1.6rem = 16px */
}
/* Typography
–––––––––––––––––––––––––––––––––––––––––––––––––– */
h1 {
	font-size: 4.0rem;
}
h2 {
	font-size: 3.6rem;
}
h3 {
	font-size: 3.0rem;
}
h4 {
	font-size: 2.4rem;
}
h5 {
	font-size: 1.8rem;
}
h6 {
	font-size: 1.5rem;
}
/* Larger than phablet */
@media (min-width: 600px) {
h1 { font-size: 5.0rem; }
h2 { font-size: 4.2rem; }
h3 { font-size: 3.6rem; }
h4 { font-size: 3.0rem; }
h5 { font-size: 2.4rem; }
h6 { font-size: 1.5rem; }
}

表示切り替え

Media Queryを使用してクラスの表示非表示を切り替えてください。

例）

「.hidePC」、「.hideSP」などで切り替える
/* スマートフォン */
.hideSP{
	display: block;
}
.hidePC {
	display: none;
}
.hideTab {
	display: none;
}

/* PC */
@media only screen and (min-width: 960px) {
.hideSP{
	display: none;
}
.hidePC {
	display: block;
}
.hidePC {
	display: block;
}
}

フルードイメージの作り方

フルードイメージは画像のサイズをブラウザのウィンドウサイズに合わせて表示する方法です。 通常、imgタグにはwidthとheightを指定するのが必須ですが、フルードイメージにする場合にはwidthとheightの指定は行いません。

/* html */
<img src="img/sample.png" alt="">
/* css */
img {
	max-width: 100%;
	height: auto;
}
